# 设想：

- 远程用户可查看食堂拥挤情况，可远程下单（即在小程序下单），但需要消耗积分，并且有距离限制（以保障菜品完成后可及时被用户拿取）
- 现场用户，可通过上报食堂拥挤程度、窗口等待时长获取积分！
- 签到、写菜品评价均可获得积分！

`API 1``API 2`

# 首页
1. 显示推荐菜品
  - 调用`API 28`，
  - 根据windowId调用`API 22`查询canteenId
2. 显示食堂列表`API 12`
  - 注意显示每个食堂的拥挤程度，以红绿灯形式（前端自行设计）
3. 点击某个食堂进入
  - 调用`API 14`显示食堂详细信息
  - 调用`API 30`以播放器形式滑动播放4道推荐菜
    - 点击可进入菜品详情`API 29`
  - 调用`API 15`显示窗口列表
  - 显示每个窗口等待等级，即waitTimeLevel（0/1/2）,等级越高等待时间越长（前端自行设计显示）
  - 上报食堂拥挤程度`API17`，获取积分！（弹出设计，底端弹出小框）
4. 点击某个窗口
  - `API 22`显示该窗口详情
    - 等待等级可转化为红黄绿颜色显示，端自行优化页面设计
    - 显示等待时长
  - 上报该窗口等待时长`API 24`，获取积分！（弹出设计，底端弹出小框）
  - 调用`API 23`显示所有菜品
    - 提供下单按钮
    - 点击可进入菜品详情`API 29`
      - 显示菜品详情`API 29`
      - 调用`API 43`显示菜品评价
      - 同样提供下单按钮
5. 下单`API 35`，扣除积分!

# 订单
1. 查看我的订单`API36`
2. 点击进入查看订单详情`API 37`
  - 提供取消订单选项（注意订单状态条件）`API 38`
  - 提供写餐后评价选项`API 42`，获取积分！（自行设计优化）
